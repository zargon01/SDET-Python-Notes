# Type Casting

**Type casting** (also known as type conversion) is the process of converting a value from one data type to another. Python provides two ways of performing type casting:

## 1. Implicit Type Casting (Automatic Conversion)

- Done **automatically** by the Python interpreter.
- Occurs when Python converts a smaller or safer data type to a larger or compatible type to **prevent data loss**.
- Usually happens during arithmetic operations when operands are of different types.

### Example of Implicit Casting
```python
num_int = 10        # int
num_float = 3.14    # float

result = num_int + num_float  # Python automatically converts int to float
print(result)      # Output: 13.14
print(type(result)) # <class 'float'>
```
In the above example, 10 (int) is implicitly converted to 10.0 (float) before addition.

## 2. Explicit Type Casting (Manual Conversion)

- Performed manually by the programmer using built-in functions.
- Also called type conversion functions.
- Useful when you need to force a specific data type.
- **Warning:** May lead to data loss or errors if not used carefully (e.g., converting float to int truncates decimal part).

### Common Explicit Type Conversion Functions

| Function | Converts To | Example | Result | Notes |
|----------|-------------|---------|--------|-------|
| `int()`  | Integer     | `int(3.9)` <br> `int("25")` | `3` <br> `25` | Truncates decimal, converts string of digits |
| `float()`| Floating-point| `float(5)` <br> `float("3.14")` | `5.0` <br> `3.14` | Adds .0 to integers |
| `str()`  | String      | `str(100)` <br> `str(3.14)` | `"100"` <br> `"3.14"` | Converts anything to string |
| `bool()` | Boolean     | `bool(0)` <br> `bool(42)` | `False` <br> `True` | 0 → False, non-zero → True |
| `list()` | List        | `list("hello")` <br> `list((1,2,3))` | `['h','e','l','l','o']` <br> `[1,2,3]` | Converts iterable to list |
| `tuple()`| Tuple       | `tuple([1,2,3])` | `(1, 2, 3)` | Converts iterable to tuple |
| `set()`  | Set         | `set([1,2,2,3])` | `{1, 2, 3}` | Removes duplicates |
| `dict()` | Dictionary  | `dict([[1,"a"], [2,"b"]])` | `{1: 'a', 2: 'b'}` | Needs iterable of key-value pairs |

### Examples of Explicit Casting
```python
# Float to Int (data loss possible)
x = 7.8
y = int(x)      # y = 7 (decimal part is truncated, not rounded)
print(y)

# String to Int/Float (common when taking input)
age = input("Enter your age: ")   # input() always returns string
age = int(age)                    # explicit conversion needed for math

# Int to String (for concatenation)
num = 42
message = "The answer is " + str(num)
print(message)
```

## When to Use Type Casting?

| Situation | Recommended Conversion | Reason |
|-----------|-------------------------|--------|
| Performing arithmetic operations | Convert to int or float as needed | Prevent TypeError |
| Taking user input (`input()`) | `int()`, `float()` on numeric input | `input()` returns string |
| Printing/converting numbers to text | `str()` | Concatenate with strings |
| Converting between collection types | `list()`, `tuple()`, `set()` | Change mutability or remove duplicates |
| Working with APIs/files | Often `str()` ↔ `int`/`float`/`bool` | Data is usually string-based |

## Best Practices & Warnings

- Always handle possible `ValueError` when converting strings:
```python
try:
    num = int(user_input)
except ValueError:
    print("Please enter a valid number!")
```
- Remember that `int("3.14")` will raise `ValueError` — use `float()` first if needed.
- `int(3.9)` gives `3`, not `4` (truncation, not rounding). Use `round()` if rounding is desired.
