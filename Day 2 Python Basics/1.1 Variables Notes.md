# Variables

## What is a Variable?
A **variable** is a named container (or memory location) used to store data in a program.  
The value stored in a variable can **change** during program execution.

**Simple Definition (Best for Exams):**  
**"A variable is a named storage location that holds a value which can be modified during program execution."**

**Example:**
```python
num1 = 10          # stores integer 10
name = "Shivam"    # stores string "Shivam"
marks = 95.5       # stores float
is_student = True  # stores boolean
```

## Why Do We Use Variables?
| Reason | Explanation |
|--------|-------------|
| 1. Store Data | Hold values like numbers, names, etc. |
| 2. Reuse Values | Use the same value multiple times |
| 3. Make Program Dynamic | Change values based on input or logic |
| 4. Perform Calculations | Use in math operations (add, subtract, etc.) |

## Rules for Naming Variables in Python
| Rule | Valid Example | Invalid Example |
|------|---------------|-----------------|
| Must start with letter or underscore (_) | name, _age | 1num, 123 |
| Can contain letters, digits, underscore | user2, roll_no | - |
| Cannot start with a number | age11 | age |
| No spaces allowed | first_name | first name |
| Case-sensitive | Name ≠ name | - |
| No special chars (!@#$%^&*) | total_marks | total@marks |
| Should not be Python keywords | count | class, for, if |

**Best Practice:** Always use meaningful names  
**Good:** student_name, total_marks, is_active  
**Avoid:** x, temp, a1

## Dynamic Typing in Python (Very Important!)
- Python is a dynamically typed language
- No need to declare data type
- Type is decided automatically at runtime

```python
age = 25        # Python understands → int
name = "Ravi"   # Python knows → string
marks = 98.6    # Automatically → float

# You can even change the type later — no error!
x = 10      # int
x = "Hello" # now becomes string
x = 3.14    # now becomes float
```

## Memory Allocation – How Variables Really Work
- Variables do not store the value directly
- They store a reference (memory address) to the actual object
- Every object has: ID, Type, and Value

```python
num = 10
print(id(num))     # Memory address
print(type(num))   # <class 'int'>
print(num)         # 10
```

## Two Types of Objects in Python
| Type | Can Change Value? | Examples |
|------|--------------------|----------|
| Immutable | No → new object created on change | int, float, str, bool, tuple |
| Mutable | Yes → value changes in same memory | list, dict, set |

## Memory Concepts
| Memory Type | What is Stored? |
|-------------|------------------|
| Heap Memory | Actual objects (values): numbers, strings, lists |
| Stack Memory | Variable names + references (pointers) to heap |

**Garbage Collection:** If no variable points to an object → Python automatically deletes it (frees memory)

## Default Values & Range/Limits
| Data Type | Default Value? | Range / Limit |
|-----------|---------------|---------------|
| Integer | No default | Unlimited (depends on memory) |
| Float | No default | ~15–17 decimal places accuracy |
| String | No default | Up to ~10 million characters (system dependent) |
| Boolean | No default | Only True or False |
